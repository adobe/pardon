Pardon is a composable dynamic templating language powering an http framework.
Templates are used to both match requests and extend them, as well as for processing responses.
 
Traditional HTTP template systems render `input` (environment value) + `"{{input}}"` template, and
Pardon supports this.  Pardon also supports `"{{input = expression}}"` (evaluating a script),
`"{{?input}}"` (for optional inputs), and `{{@input}}` for handling secrets.

## Example

To demonstrate Pardon's core features, we'll describe a simple multi-user TODO REST service:

- This service is theoretically hosted at `https://todo.example.com/`.
- It has endpoints to manage users, supports exchanging password-based authentication for
a bearer token.
- Logged in users can edit their todo list (create, read, update, delete, and list). 
- Todo objects have a task (text description) and a "done" flag.

Please familiarize yourself with the following 8 requests to manage users, create and list todos,
and update those todos.

A pardon collection would contain the above templates in `.https` files,
or "HTTP-schema" files.
{/* Yes, I'm quite proud of myself for that name, thanks for asking */}
Schema files have a little more syntax:

A schema file has one (or more) variants for making a request
prefixed by `>>>` for each outbound request,
and optionally, templates to match and process the response
prefixed by `<<<` for each inbound response.

Response templates can extract and redact values in a response,
for example, the user login HTTP schema can specifies a value
placeholder in the response, `{{@token}}`.

```http
>>>
PUT https://todo.example.com/users
Content-Type: application/json
Authorization: User {{@user-token}}

{
  "username": "{{username}}",
  "password": "{{@password}}"
}

<<<
200 OK

{
  "token": "{{@token}}"
}
```

A script can use the extracted value when using `pardon()`'s
Javascript API to access the authorization call.
Here's an example of a script that makes another request.

```ts
import { pardon } from "pardon";

export async function authorizeUser(username: string) {
  const password = await getPassword(username);

  const {
    inbound: {
      secrets: { token },
      response: res,
    },
  } = await pardon({ username, password })`
    PUT https://todo.example.com/users
  `();

  if (!token) {
    throw new Error(
      `failed to authorize user ${username}: (${res.status}) ${res.body}`,
    );
  }

  return token;
}
```

We can import this function and define the header
`Authorization: User {{@user-token = authorizeUser(username)}}`
to "automatically" render the authorization token. (Pardon provides both in-memory, and persisted
caching options as well.)

 - create a todo item.
 - mark the todo as completed.
